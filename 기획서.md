# 앤딩 스터디카페 환불 계산기 서비스 기획서

## 요약
- **배경**: 앤딩 스터디카페 점주들은 이용권별 환불 규정이 복잡하여 기존에 엑셀로 환불 금액표를 관리해왔음
- **원인**: 엑셀 파일은 모바일 접근이 불편하고, 이용권 종류마다 환불 조건이 상이하여 현장에서 즉각적인 고객 안내가 어려웠음
- **해결**: 모바일 특화 웹 앱을 개발하여 드롭다운 선택과 일수/시간 입력만으로 환불금액을 자동 계산하고 즉시 표시
- **기대**: 고객 환불 안내 시간을 수 분에서 수 초로 단축하고, 계산 오류 없이 정확한 금액을 모든 점주가 공유
- **운영**: GitHub 연동 자동 배포로 환불 규정 변경 시 즉시 반영, UptimeRobot으로 상시 접속 가능 유지

## 2. 도입 취지

앤딩 스터디카페 점주들은 기간권·시간권의 환불 규정이 이용권 종류별로 복잡하게 상이하여, 기존에는 엑셀로 환불 금액표를 정리해 운용하였다.

그러나 아래와 같은 문제가 지속적으로 발생하였다.

- **모바일 접근성 부재**: 엑셀 파일은 PC 환경에 최적화되어 있어 현장에서 스마트폰으로 빠르게 확인하기 어려움
- **규정의 복잡성**: 기간권(주차별 차등 환불), 시간권(시간당 차감), 당일권·스터디룸(환불 불가) 등 이용권 유형별 환불 조건이 상이하여 즉각적인 안내가 어려움
- **고객 응대 어려움**: 환불 요청 고객에게 정확한 금액을 즉시 산출·안내하지 못해 신뢰도 저하 및 분쟁 발생 가능성 존재

이에 따라 **모바일에서 드롭다운 선택만으로 환불금액을 즉시 계산**하는 웹 앱을 개발·배포하여, 모든 점주가 언제 어디서나 정확한 환불금액을 확인하고 고객에게 안내할 수 있도록 한다.

## 3. 문제 정의 및 해결 방향

### 문제
```
엑셀 파일
  → 모바일 열기 불편
  → 이용권별 시트 찾기 번거로움
  → 계산 직접 해야 함
  → 고객 대기 시간 증가
  → 환불 금액 오류 가능성
```

### 해결
```
모바일 웹 앱
  → 브라우저에서 즉시 접속
  → 드롭다운으로 이용권 선택
  → 사용일수/시간 입력 → 자동 계산
  → 고객에게 즉시 안내
  → 오류 없는 정확한 금액 산출
```

## 4. 주요 기능

### 4-1. 기간권 환불 계산
- 이용권 종류 선택 (2주권 / 4주권 / 고정석 4주권 / 관리고정석 4주권 / 오픈행사 4주권)
- 사용 일수 입력
- 주차 자동 판별 → 환불 가능 여부 및 환불금액 즉시 표시

| 상품명 | 상품금액 | 1주차 | 2주차 | 3주차 | 4주차~ |
|--------|---------|-------|-------|-------|--------|
| 2주권 | 55,000원 | 22,000원 | 불가 | — | — |
| 4주권 | 119,000원 | 76,300원 | 43,600원 | 21,800원 | 불가 |
| 고정석 4주권 | 179,000원 | 125,300원 | 71,600원 | 35,800원 | 불가 |
| 관리고정석 4주권 | 229,000원 | 160,300원 | 91,600원 | 45,800원 | 불가 |
| 오픈행사 4주권 | 79,000원 | 55,300원 | 31,600원 | 15,800원 | 불가 |

### 4-2. 시간권 환불 계산
- 이용권 종류 선택 (100시간권 / 200시간권 / 오픈100시간권)
- 이용시간 및 구매 후 경과 일수 입력
- 환불금액 = 상품금액 − (이용시간 × 1,500원), 28일 초과 시 환불 불가

| 상품명 | 상품금액 |
|--------|---------|
| 100시간권 | 95,000원 |
| 200시간권 | 180,000원 |
| 오픈100시간권 | 69,000원 |

### 4-3. 확장 기간권 환불 계산 (2~20주권)
- 주권 기간 및 상품금액 직접 입력
- 전체 기간을 4분기로 균등 분할하여 비율 자동 적용
  - 1분기: 70% / 2분기: 40% / 3분기: 20% / 4분기: 환불 불가

### 4-4. 환불 규정 전체 보기
- 전체 이용권 환불 규정 한눈에 조회

```

    ↓ 브라우저 접속

    ↓
app.py  ──→  refund_calculator.py (계산 로직)
```
